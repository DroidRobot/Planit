## Testing

### Overview

This section documents the unit tests and integration tests generated for the functions implemented in PM4. The tests follow the course AI policy and are generated with AI assistance, with prompts documented below.

### AI Disclosure for Test Generation

**AI Tool Used:** Claude Code (Anomally)

**Prompt:**
> Generate unit tests using Jest and supertest for the tasks.js Express route file at backend/src/routes/tasks.js. The tests should cover: GET /api/tasks (list tasks for a plan), POST /api/tasks (create task), PUT /api/tasks/:id (update task), POST /api/tasks/reorder (bulk reorder), DELETE /api/tasks/:id (delete task). Use mocking for the database pool. Include tests for authentication requirements, validation errors, and successful operations.

> Also generate integration tests that test a complete user workflow: signup -> create plan -> add tasks -> mark tasks complete -> delete plan.
